# SNMP 가이드

> OWLmon에서 네트워크 장비를 모니터링하는 방법과 SNMP 개념 정리

---

## SNMP란?

**Simple Network Management Protocol** — 네트워크 장비를 원격으로 모니터링하는 표준 통신 규약 (1988년 표준, 거의 모든 네트워크 장비 지원)

### 왜 필요한가?

서버/PC는 OWLmon 에이전트를 설치할 수 있지만,
**스위치, 라우터, 프린터**는 OS가 없어서 에이전트 설치가 불가능하다.

이 장비들이 공통으로 지원하는 "외부에서 상태를 물어볼 수 있는 기능"이 SNMP다.

```
OWLmon 서버 ──── 질문 (포트 161/UDP) ────▶ 스위치/라우터
             ◀─── 답변 (트래픽, 상태 등) ───
```

---

## 핵심 개념

### Community String

비밀번호 역할을 하는 문자열.

```
서버: "비밀번호: public. 트래픽 알려줘"
장비: "현재 포트 1번 100Mbps, 포트 2번 DOWN"
```

- 기본값이 대부분 `public`
- **평문 전송** — 암호화 없음 (v2c의 최대 단점)
- 같은 네트워크에서 패킷 캡처하면 노출됨

### OID (Object Identifier)

장비에서 읽을 수 있는 데이터 항목마다 고유 번호가 있다.

| OID | 의미 |
|-----|------|
| `1.3.6.1.2.1.1.5.0` | 장비 이름 (sysName) |
| `1.3.6.1.2.1.1.3.0` | 가동 시간 (sysUpTime) |
| `1.3.6.1.2.1.2.2.1.8.{N}` | N번 포트 상태 (1=UP, 2=DOWN) |
| `1.3.6.1.2.1.2.2.1.10.{N}` | N번 포트 수신 바이트 |
| `1.3.6.1.2.1.2.2.1.16.{N}` | N번 포트 송신 바이트 |

## 버전 비교

| 항목 | v2c | v3 |
|------|-----|----|
| 인증 | Community String (평문) | 아이디 + 비밀번호 |
| 암호화 | 없음 | AES/DES |
| 보안 | 취약 | 안전 |
| 설정 난이도 | 쉬움 | 복잡 |
| 사용 권장 | 내부 테스트, 소규모 | 공공기관 납품 필수 |

> **OWLmon 현재**: v2c 구현 완료 / v3는 추후 추가 예정

## OWLmon에서 수집하는 정보

| 항목 | 설명 |
|------|------|
| 장비 온/오프라인 | SNMP 응답 여부 |
| 가동 시간 | 마지막 재부팅 이후 경과 시간 |
| 포트별 상태 | UP / DOWN |
| 포트별 수신 트래픽 | RX bytes/sec |
| 포트별 송신 트래픽 | TX bytes/sec |

## 실제 사용 예시 (학교 환경)

```
[인터넷]
    │
[방화벽/라우터]  ← SNMP로 모니터링 (전체 트래픽 현황)
    │
[코어 스위치]   ← SNMP로 모니터링 (포트별 상태)
    ├── 교무실 스위치  ← SNMP로 모니터링
    ├── 행정실 스위치  ← SNMP로 모니터링
    └── 전산실 스위치  ← SNMP로 모니터링
```

OWLmon에서 볼 수 있는 것:
- 전산실 → 교무실 구간 포트 DOWN → 교무실 인터넷 끊김 원인 파악
- 특정 시간대 트래픽 급증 → 누가 대용량 파일 전송하는지 파악
- 스위치 재부팅 감지 (가동 시간 초기화)

## 장비 등록 방법

### 1. 장비에서 SNMP 활성화

장비마다 다르지만 일반적으로:
- 관리 웹페이지 접속 → SNMP 설정
- Community String 설정 (읽기 전용으로만 설정 권장)
- OWLmon 서버 IP를 허용 목록에 추가

**pfSense**: Services > SNMP > Enable
**일반 스위치**: 제조사별 관리 페이지에서 SNMP 활성화

### 2. OWLmon 대시보드에서 등록

상세 보기 탭 → 네트워크 장비(SNMP) 섹션 → 장치 추가

| 항목 | 설명 |
|------|------|
| 장치 이름 | 식별용 (예: 교무실 스위치) |
| IP 주소 | 장비 관리 IP |
| Community String | 장비에 설정한 값 (보통 `public`) |
| 포트 | 기본 161 |

---

## 보안 주의사항

- `public` Community String은 변경 권장 (추측하기 쉬움)
- SNMP 포트(161/UDP)는 OWLmon 서버 IP만 허용
- 공공기관 납품 시 v3 사용 (암호화 + 인증)
- 개인 PC에 SNMP 서비스를 열어두지 말 것
