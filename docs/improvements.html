OWLmon 개선 사항

개발 중 발견된 문제, 개선 내용, 향후 과제 정리

완료된 개선

host_name 레이블 누락 수정 완료
인프라 · OTel Collector 설정
OTel Collector가 에이전트의 리소스 속성(host.name)을 Prometheus 레이블로 변환하지 않는 문제. infra/otel/otel-collector.yml에 resource_to_telemetry_conversion: enabled: true 추가로 해결. 다중 호스트 구분에 필수.
Collector 재시도 설정 추가 완료
에이전트 · exporter/otlp.go
Collector 일시 다운 시 데이터가 즉시 유실되는 문제. OTLP exporter에 retry 설정 추가: 초기 간격 5초, 최대 간격 30초, 최대 재시도 5분.
인메모리 버퍼링 구현 완료
에이전트 · exporter/buffered.go
Collector 연결 실패 시 메트릭 데이터를 메모리에 보관(최대 100개 배치, 약 50분치)하다가 재연결 시 자동 전송. 버퍼 초과 시 가장 오래된 데이터부터 제거.

알려진 한계

에이전트 자체가 꺼진 동안의 데이터는 복구 불가능합니다. 버퍼링은 에이전트가 살아있고 Collector만 다운된 경우에만 유효합니다. 이는 모든 Push 모델 모니터링의 근본적 한계입니다.
에이전트 다운 시 데이터 공백 근본적 한계
에이전트가 꺼진 구간은 데이터 없음
Prometheus 기본 만료 시간(5분) 이후에도 에이전트가 재연결 안 되면 시리즈 만료. 재시작 후 30초 이내 새 데이터 수신 시 자동 복구됨.
버퍼 영속성 없음 향후 과제
에이전트 재시작 시 버퍼 데이터 소실
현재 인메모리 버퍼라 에이전트 재시작 시 버퍼 초기화됨. 파일 기반 버퍼(SQLite 또는 WAL 파일)로 개선 시 재시작 후에도 미전송 데이터 유지 가능.

향후 과제

파일 기반 영속 버퍼 예정
에이전트 재시작 후에도 미전송 데이터 보존
SQLite 또는 WAL 파일에 버퍼 저장. 에이전트 재시작 시 이전 버퍼 복구 후 전송.
HTTP/TCP 서비스 체크 예정
웹사이트, 포트 응답 감시
HTTP 응답코드, 응답시간 / TCP 포트 열림 여부 / SSL 인증서 만료 감지.
알림 연동 예정
임계값 초과 / 서버 다운 알림
카카오톡 알림톡, 슬랙, 이메일. 디바운싱 + 중복 제거 + 심각도 분류 필수.
로그인 / 인증 예정
현재 누구나 대시보드 접근 가능
API 키 또는 JWT 기반 인증. 멀티테넌트 대비 설계.
