# OWLmon 개발 로드맵

> 타겟: 학교 / 공공기관 (IT 전담 없는 중소 규모)
> 경쟁 타겟: Zabbix 대비 UX·자동화·한국어 차별화

---

## 현재 구현 완료

| 기능 | 설명 |
|------|------|
| 서버 에이전트 | CPU / 메모리 / 디스크 / 네트워크 I/O 수집 (Go, gopsutil) |
| 서비스 체크 | TCP 포트 / HTTP 응답 상태 수집 |
| 멀티 호스트 | 여러 서버 동시 모니터링, 탭/Overview 전환 |
| 이메일 알림 | 임계값 초과 / 회복 알림 (SMTP) |
| 알림 히스토리 | PostgreSQL 저장, 대시보드 조회 |
| 월간 보고서 | 호스트별 avg/max/uptime 집계, 이메일 자동 발송 (매월 1일) |
| 인증 | JWT 로그인, HTTPS 지원 |
| Windows 서비스 | 서버/에이전트 서비스 등록 지원 |
| 개발 환경 | dev.ps1 원클릭 실행, 크래시 자동 재시작 |

## 우선순위 로드맵

### 1순위 — 납품 필수 요건

#### SNMP 네트워크 장비 모니터링
- **왜**: 학교는 반드시 스위치/라우터가 있음. 서버만 보는 모니터링은 반쪽짜리
- **내용**: IP + Community String 등록 → 포트별 트래픽 / 장비 상태 수집
- **구현**: gosnmp 라이브러리, 서버에서 폴링 → Prometheus 노출

#### 에이전트 버퍼링
- **왜**: 네트워크 끊겼을 때 데이터 유실 → 감사에서 문제됨
- **내용**: 에이전트가 오프라인 시 로컬에 데이터 임시 저장, 재연결 시 일괄 전송
- **구현**: 에이전트 내 로컬 큐 (메모리 or 파일)

---

### 2순위 — 차별화 포인트

#### 예측 알림 (디스크 용량)
- **왜**: 학교 예산은 연 단위. "3개월 후 부족 예상" → 사전 구매 요청 가능
- **내용**: Prometheus `predict_linear()` 활용, N일 후 예상 시점 계산 후 알림
- **구현**: 알림 체커에 예측 쿼리 추가

#### SSL 인증서 만료 알림
- **왜**: 학교 홈페이지 인증서 만료 = 민원. 담당자가 놓치는 경우 많음
- **내용**: 도메인 등록 → 만료 30일/7일 전 이메일 알림
- **구현**: crypto/tls로 인증서 조회, 에이전트 또는 서버에서 주기적 체크

#### 모바일 최적화
- **왜**: IT 담당자가 퇴근 후/주말에 알림 받고 확인해야 할 때 PC가 없음
- **내용**: 반응형 레이아웃, PWA 지원 (홈 화면 추가)
- **구현**: CSS 미디어 쿼리, vite-plugin-pwa

---

### 3순위 — 완성도

#### 서버 추가 온보딩 UI
- **왜**: 현재 에이전트 설치가 수동. 납품 시 고객이 직접 못 함
- **내용**: 대시보드에서 에이전트 설치 스크립트 생성 / 안내

#### 로그 수집
- **왜**: 공공기관은 보안 감사 대응용 로그 보관 의무
- **내용**: 에이전트에서 시스템 이벤트 로그 수집, 대시보드 조회

#### AI 자연어 보고서 요약
- **왜**: 교장/행정실장에게 월간 보고 시 숫자보다 글이 전달력 있음
- **내용**: Claude API 연동, 월간 통계를 자연어로 변환
- **예시**: "이번 달 서버 A는 매주 화요일 오전 CPU가 급등하는 패턴이 반복됐습니다. 디스크는 약 3개월 후 증설이 필요합니다."

| SNMP | 강력 | (개발 중) |
