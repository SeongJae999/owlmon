# Docs Skill

현재까지 진행한 작업을 기반으로 `docs/` 디렉토리에 문서를 작성하거나 업데이트합니다.

## 수행 절차

1. 현재 대화 맥락과 작업 내용 파악
2. 관련 기존 문서가 있으면 업데이트, 없으면 새로 생성
3. 아래 원칙에 따라 작성

## 작성 원칙

- **왜 이걸 쓰는지** 동기 먼저 설명 (기술 선택 이유 포함)
- **처음 보는 사람도 따라할 수 있게** 단계별로
- 한국어 기본, 기술 용어는 영문 병기
- 코드 예시 포함 시 한국어 주석 충분히
- 개념 설명 시 비유나 다이어그램 활용

## 문서 종류 및 저장 위치

| 종류 | 경로 | 내용 |
|------|------|------|
| 기술스택 | `docs/TECH-STACK.md` | 기술 선택 이유, 비교 |
| 아키텍처 | `docs/ARCHITECTURE.md` | 시스템 구조, 데이터 흐름 |
| 개발 가이드 | `docs/guides/` | 설치, 실행, 개발 방법 |
| ADR | `docs/decisions/` | 아키텍처 의사결정 기록 |
| API 스펙 | `docs/API.md` | 엔드포인트 정의 |
